Javascript面向对象的程序设计

2020-03-01

理解对象

ES5中规定对象有两种属性，数据属性和访问器属性，每一个对象的属性都有一个描述对象（Descriptor），用于控制该对象的行为。
使用Object.getOwnPropertyDescriptor和Object.getOwnPropertyDescriptors可以获取到该描述对象

  1. 数据属性
  • [[Writable]]：表示能否修改属性的值，默认为true
  • [[value]]：包含这个属性的值，默认为undefined
  • [[Configurable]]表示能够通过delete删除属性从而重新定义属性，能否修改属性的描述对象，默认为true。一旦将这个属性设置为false就无法重置为true。
  • [[Enumerable]]表示可枚举性，默认为true，如果这个设置为false，有一些操作会访问不到这个属性
  • for in：只遍历对象自身和继承的可枚举属性
  • Object.keys：返回对象自身的所有可枚举属性
  • JSON.stringify：只串行化自身的可枚举属性
  • Object.assign：会忽略

引入enumerable就是为了不让forin遍历某些属性，比如数组的length，对象原型的toString方法。如果只考虑自身的属性，尽量不要用for in循环

ES6规定class的原型方法都不可枚举。

Object.defineProperty和defineProperties

通过这两个方法可以修改一个属性的描述对象，在调用该方法创建一个新属性时，如果不指定，writable、enumerable、configurable默认为fasle

  1. 访问器属性
    访问器属性不包括数据值，它们有一对getter和setter函数（都可选），读取该属性时会调用getter函数，返回函数的结果，写入属性时会调用setter函数，有以下四大特性* [[Configurable]]表示能够通过delete删除属性从而重新定义属性，能否修改属性的描述对象，默认为true。一旦将这个属性设置为false就无法重置为true。
  • [[Enumerable]]
  • [[Configurable]]
  • [[Get]]：默认值为undefined
  • [[Set]]：默认值为undefined

这个方法是vue的核心所在，可以创造出惊人的效果。

原型方法扩展

__proto__属性是一个非标准属性，只有浏览器要求部署，ES6提出了三个新的API

  • Object.setPrototypeOf
  • Object.getPrototypeOf
  • Object.create

Object.setPrototypeOf的效率比Object.create的效率要低

继承

基于原型链实现继承

每一个构造函数都有一个原型对象，原型对象都包含一个指向构造函数的指针，而实例都包含一个指向原型对象的内部指针。

假如我们让原型对象等于另外一个构造函数的实例，此时原型对象将包含一个指向另外一个原型的指针。层层递进这就是原型链的概念。

把SubType的原型对象改写为SuperType的实例，实现原型链的连接，也就是

当读取一个实例的属性的时候，会沿着原型链往上查找

  1. 搜索自身
  2. 搜索构造函数的原型对象
  3. 搜索构造函数的原型对象的原型
  4. 直到原型链的尽头

缺点：

  • 超类的实例属性如果有引用类型会被共享，比如一个数组
  • 继承时，如果向超类传递参数会影响其他实例

借用构造函数

就是在子类中调用超类的方法。

无法函数进行函数复用，一般不单独使用

组合继承

原型链加借用构造函数，使用原型链实现对原型属性和方法的继承，通过借用构造函数来实现对实例属性的继承，组合继承调用了两次父构造函数，
生成了两个同名属性，只是实例属性将原型上的属性屏蔽了。

原型式继承

这个跟原型模式差不多，会共享引用类型。

ES5的Object.create就是基于这个思想实现的，接受一个作为新对象的原型的对象和一个为新对象定义额外属性的对象（可选）

寄生式继承

寄生式组合式继承

寄生组合式继承的基本思想是为了解决组合继承的缺点，组合继承调用了两次父构造函数，生成了两个实例属性，只不过实例上的覆盖了原型上的属性。用了寄生式继承的方法，将子类原型指向父类原型

实际上就是省略了组合继承的SubType = new SuperType这一步，减少了一次调用构造函数，效率高一些。
