《ES6标准入门》学习笔记第11章 Set和Map数据结构

2020-02-03

11.1 Set

Set是类数组结构，但成员都是唯一的，没有重复

在set内部NaN是相等的，与===不相同

11.1.2 Set的属性和方法：

  • Set.prototype.size：返回实例的成员总数
  • add(val)：返回实例本身
  • delete(val)：返回boolean值
  • has(val)：返回boolean值
  • clear(val)：清除所有值，没有返回值

11.1.3 Set的遍历操作

  • keys()：返回键名的遍历器
  • values()：返回键值的遍历器
  • entries()：返回键值对的遍历器
  • forEach()：使用回调函数遍历每个成员

set的遍历顺序就是插入的顺序

set没有键名，或者说键名和键值相等，所以keys和values方法的行为完全一致

11.2 WeakSet

WeakSet跟Set类似，主要有两点区别

  1. WeakSet只能存放对象
  2. WeakSet中的对象都是弱引用，也就是垃圾回收机制会忽略它对对象的引用，所以WeakSet的成员个数是不能确定的，所以不能遍历

WeakSet接受一个具有iterable接口的对象作为参数，该对象的成员会成为WeakSet实例的成员，而不是该对象本身

WeakSet拥有以下三个方法：

  • WeakSet.prototype.add(val)
  • WeakSet.prototype.delete(val)
  • WeakSet.prototype.has(val)

由于WeakSet的成员随时都可能消失，可以利用这个特点来存储DOM节点，而不用担心这些节点被移除时发生内存泄漏

11.3 Map

JS中的对象只能用字符串作为键，Map可以使用任何值作为键，是一种更加完善的hash结构

Map的实例方法和遍历方法和Set的类似，同时Map的遍历顺序就是插入的顺序

Map的一个应用

leetCode 146. LRU缓存机制
因为Map的遍历顺序是按照插入的顺序确定的，所以可利用这个特点模拟实现LRU

11.4 WeakMap

WeakMap跟WeakSet类似，也只接受对象作为键，所指向的对象不计入垃圾回收机制

WeakMap的用途
用于作为监听事件的listener对象
